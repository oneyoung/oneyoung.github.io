Pyhton Naming Conventions

Package & Module

  • 名字尽量短, 全部使用小写
  • Module 在必要时可以使用下划线_, 而package则不建议.
  • 用C或者C++扩展的Module, 在前面加_(比如:_socket)

Class & Exception

  • 统一使用CapWords
  • 只在内部使用的类, 前面加_(比如_CapWords)
  • 对于Exception, 如果确实是错误的话, 加Error后缀

Constants

  • 全部大写, 以下划线分词(比如MAX_OVERFLOW)

Function & Global Variables

  • 全部小写, 以下划线分词(比如joined_lower
  • mixedCase只有在为兼容先前代码才可使用
  • 全局变量应使用__all__机制, 避免在from M import *时, 被导出

Method & Attribute

  • method的规则和function一样
  • 始终将self作为instance method的第一个参数名, cls作为class method的第一个参数名
  • Attributes: interface, _internal, __private

Reference:

  1. PEP8 -- Style Guide for Python Code
  2. Code Like a Pythonista: Idiomatic Python
